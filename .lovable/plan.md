
# UX-polish ROSA&BAROCCO (без архитектурных изменений)

Цель: ускорить путь к покупке и снять визуальный шум. Без новых библиотек, без бэкенда, без изменений роутинга.

## 1. Главная — hero

`src/routes/index.tsx`
- Сократить H1 на mobile: «Натуральные коллоидные растворы серебра, золота и платины».
- Сократить под-текст до одной строки про спрей-мист.
- Под CTA добавить trust-строку: «Бесплатная доставка от 1500 ₽ · Натуральный состав · Доставка по России».
- CTA «Перейти в каталог» → «Открыть каталог».
- Mobile: `py-12` → `py-10`.

## 2. Карточки товаров

`src/components/catalog/ProductCard.tsx`
- Над названием — статичная мини-метка категории (Платина / Золото / Серебро / Набор) вместо hover-«Подробнее».
- Цена крупнее и плотнее: `text-[15px] font-semibold`.
- Добавить icon-only кнопку «+» в правом нижнем углу карточки (`e.preventDefault()` + `add(product)` + toast) — 1 тап до добавления, без перехода в карточку товара.

## 3. Карточка товара — sticky bar

`src/routes/product.$slug.tsx`
- Уменьшить `py-3` → `py-2.5` у sticky bar.
- Добавить тонкую верхнюю тень для визуального отделения.
- Текст в кнопке: «В корзину · {цена}».

## 4. Корзина

`src/routes/cart.tsx`
- На mobile (≤ lg) добавить sticky bottom bar с итогом и кнопкой «Оформить заказ», чтобы не скроллить вниз.
- Добавить `pb-28 lg:pb-0` к контейнеру.
- Empty state — добавить второй link «Посмотреть бестселлеры».

## 5. Checkout — упростить

`src/routes/checkout.tsx`
- Email сделать опциональным (`z.string().email().optional().or(z.literal(""))`). Менеджер связывается по телефону.
- Объединить «Контактные данные» и «Доставка» в один блок (один `Section`), меньше визуальных рамок.
- Под каждым radio способа доставки — короткий hint: «350 ₽» / «по тарифу» / «по тарифу».
- Чекбокс: «Согласен с обработкой персональных данных».
- CTA «Подтвердить заказ» → «Оформить заказ».
- На mobile: sticky bottom submit bar с итогом + кнопкой; обычная кнопка в aside остаётся для desktop.

## Чего НЕ делаем

- Не меняем архитектуру, роутинг, zustand-стор, типы данных, seed, шрифты, цветовые токены, изображения.
- Не добавляем пакеты, backend, Supabase, оплату, мессенджеры, CRM.
- Не переписываем формы (только тексты и опциональность email/comment).
